# NeurIPS 审稿意见 — FinAgent-Evo

> 审稿日期：2026-04-18
> 论文版本：neurips_paper/main.tex（commit a388763）

---

## 论文摘要

FinAgent-Evo 提出了一个金融 LLM 智能体框架，结合可进化的技能提示、层次化记忆与程序规则抽象、以及 DAG 风格的验证执行。在 20 个任务的真实 API 基准和 FinanceReasoning 上进行了评估。

---

## 优点

1. **设计分解清晰。** 三组件设计（进化 + 记忆 + DAG 编排）动机充分，每个消融实验都能干净地对应一个组件。
2. **双重成功指标。** 同时报告硬成功率（执行可靠性）和评判成功率（输出质量），方法论上合理，避免了单一指标被刷分。
3. **可复现性努力。** 固定工具 ID、时间点缓存、固定评判配置、Bootstrap 置信区间——在智能体论文中较为罕见，值得肯定。
4. **局限性部分诚实。** 直接承认了消融不完整、评判偏差、工具非确定性等问题。

---

## 缺陷

### 严重问题

1. **摘要与结果数字不一致。** 摘要报告评判成功率为 **11.7% ± 7.5%**，但表 1 中完整系统为 **81.4% ± 7.5%**。这是提交前必须解决的重大事实错误。
2. **基准规模太小。** N=20 个任务，R=3 次试验。95% Bootstrap CI 跨度达 ±5–8%，统计功效不足以支撑强结论。至少需要说明为何 N=20 足够，或扩大规模。
3. **无人工评估。** 评判模型仅为单一 LLM（Qwen3.6-plus），未报告评判者间一致性。论文承认了 LLM 评判偏差，但未量化。
4. **进化引擎未单独验证。** 消融显示去掉进化后评判成功率从 81.4% 降至 55.2%，但没有分析：实际有多少技能发生了进化、多少突变被接受/拒绝、回放缓冲区在 N=10 下是否有意义。

### 较重要问题

5. **FinBen 结果（28.05%）弱且缺乏对比。** 没有基线对比，无法判断好坏。普通 GPT-4o 能达到多少？
6. **InvestorBench 负 Alpha。** 所有资产相对买入持有策略均为负 Alpha（均值 -2.61%），这一结果被埋在表格里没有讨论。
7. **"去掉记忆"的硬成功率反而更高（98.2% vs 95.0%）。** 违反直觉且未作解释，记忆注入可能引入了不稳定性。
8. **LaTeX 正文中使用了 Markdown 加粗语法**（`**93.70\%**`、`**28.05\%**`）——PDF 中会渲染为字面星号，提交前必须修复。

### 次要问题

9. 缺少进化提示优化相关工作的引用（EvoPrompt、OPRO 等）。
10. "未来工作"小节放在了结果章节（§4.5）内，应移至结论或单独章节。
11. `\author{}` 为空——应确认 neurips_2024 样式正确处理了匿名审稿。
12. 脚注中写的递归限制为"25 步"，但敏感性分析表格中 25 也是测试值之一——需明确主实验使用的是哪个限制。

---

## 总体建议

**边界拒稿 / 大修。** 核心思路扎实，工程细节认真，但摘要与结果的数字矛盾是致命错误，基准规模不足以支撑论文声明，且若干结果需要解释而非沉默。

### 提交前优先修复

1. 修正摘要中的评判成功率数字（11.7% vs 81.4%）
2. 修复 LaTeX 加粗语法（`**...**` → `\textbf{...}`）
3. 为 FinBen 添加基线对比
4. 讨论负 Alpha 和记忆/硬成功率异常
5. 报告进化引擎统计数据（突变接受率、技能更替情况）
